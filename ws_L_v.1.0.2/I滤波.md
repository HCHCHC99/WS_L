## 2. 滤波器：2 阶 Butterworth IIR

### 设计参数

| 参数 | 值 |
|------|-----|
| 类型 | Butterworth IIR, Direct Form I |
| 阶数 | **2** |
| 采样率 fs | **50 kHz**（ISR 频率） |
| 截止频率 fc | **200 Hz**（-3dB） |
| 衰减斜率 | **-40 dB/dec** |
| 设计工具 | MATLAB: `[b,a] = butter(2, 200/25000, 'low')` |
| 运算 | float32, Cortex-M4F FPU 硬件加速 |

### 系数

```c
#define BIQUAD_B0  0.0001551484f
#define BIQUAD_B1  0.0003102968f
#define BIQUAD_B2  0.0001551484f
#define BIQUAD_A1  (-1.9644605802f)
#define BIQUAD_A2  0.9650811739f
```

### 差分方程

```
y[n] = b0.x[n] + b1.x[n-1] + b2.x[n-2] - a1.y[n-1] - a2.y[n-2]
```

### 状态初始化

首帧采样时将所有状态变量（x[n-1], x[n-2], y[n-1], y[n-2]）种子为当前采样值，消除 ramp-up 延迟，实现瞬时锁定。

## 5. 设计过程详解

### 5.1 问题分析：为什么要滤波

BLDC 电机控制中，ADC 采样到的电流信号包含三种成分：

```
采样信号 = 电机基波电流 + 换相谐波 + PWM 开关噪声
          (0~300Hz)      (1~5kHz)      (50kHz, 峰峰值可达几百 mV)
```

**目标**：保留基波电流（用于波形观察和控制），抑制 PWM 噪声和谐波。

这是一个典型的**低通滤波**问题——让低频通过，阻止高频。

### 5.2 滤波器类型选择：IIR vs FIR

数字滤波器分两大类：

| | IIR（无限脉冲响应） | FIR（有限脉冲响应） |
|------|------|------|
| 结构 | 有反馈回路 | 纯前馈 |
| 差分方程 | y[n] = sum(b_k.x[n-k]) - sum(a_k.y[n-k]) | y[n] = sum(b_k.x[n-k]) |
| 同等衰减所需阶数 | **2 阶** | 200+ 阶 |
| 相位 | 非线性 | 线性 |
| 计算量 | 极低 | 高 |
| 内存 | 极少 | 大（需要大的延迟线） |

**选择 IIR 的理由**：
- 我们只需要看电流波形的**幅值形状**，不关心相位是否线性（示波器看波形也不需要线性相位）
- 2 阶 IIR 就能达到 -40dB/dec 衰减，FIR 需要 200+ 阶才能等效
- ISR 运行在 50kHz，计算效率是第一优先级

### 5.3 Butterworth 滤波器原理

IIR 有多种类型（Butterworth、Chebyshev、Elliptic 等），区别在于**极点位置**如何放置。

**Butterworth 的特点**：所有极点均匀分布在 s 平面的单位半圆上，
使得**通带内幅频响应最平坦**（"maximally flat"）。

```
幅频响应 |H(f)| = 1 / sqrt(1 + (f/fc)^(2N))

其中: N = 阶数, fc = 截止频率(-3dB点)
```

- N=1 时在 fc 处从 0dB 开始以 -20dB/dec 滚降（这就是旧 EMA 的行为）
- N=2 时通带内保持平坦到接近 fc，然后以 **-40dB/dec** 滚降
- N 越大滚降越陡，但通带内可能出现过冲（Butterworth 不会）

为什么不用 Chebyshev / Elliptic？

| 类型 | 通带 | 过渡带 | 缺点 |
|------|------|------|------|
| **Butterworth** | **最平坦，无纹波** | 中等 | 需要稍低 fc 来补偿过渡带 |
| Chebyshev I | 有纹波 | 更窄 | 通带纹波会扭曲电流波形 |
| Elliptic | 有纹波 | 最窄 | 通带和阻带都有纹波 |

BLDC 电流基波是几十 Hz 的正弦波——**波形形状就是我们要观察的对象**。
通带纹波会直接在正弦波上叠加畸变，所以 Butterworth 是唯一正确的选择。

### 5.4 截止频率 fc 的确定

三个约束条件共同决定 fc：

**约束 1 — 保留电机基波**：
```
BLDC 电频率 = RPM x 极对数 / 60
典型范围: 1000~6000 RPM, 4 对极 -> 67~400 Hz
```
fc 必须高于电机最高工作频率。选 200Hz 意味着 3000RPM(4对极=200Hz)
处刚好在 -3dB 点，对大多数工况通带足够。

**约束 2 — 衰减 PWM 噪声**：
```
PWM 频率 = 50 kHz（开关管的基频）
fc=200Hz 时: 50kHz / 200Hz = 250 倍 = 2.4 个十倍频程
2 阶衰减: 2.4 x 40 = 96 dB
```
96dB 意味着 PWM 纹波被压到原始值的 **1/63000**，即使原始纹波
峰峰值 500mV，滤完只剩 ~0.008mV，远低于 ADC 的 1 个 LSB(0.8mV)。

**约束 3 — 阶跃响应不能太慢**：
```
fc 越低 -> 响应越慢 -> 启动/换相时波形形状滞后
fc=200Hz 时 2 阶 Butterworth 的 95% 建立时间约 12ms
控制环路在 50kHz 运行，12ms 相当于 600 个采样周期——完全可接受
```

> 如果电机长期运行在高转速（6000RPM+），可将 fc 提高到 300~400Hz。
> 如果只关心低频趋势，可将 fc 降到 100Hz。

### 5.5 MATLAB 设计步骤

#### Step 1 — 确定规格

```
fs  = 50000;   // 采样率 (ISR 执行的频率)
fc  = 200;     // 截止频率 (-3dB)
N   = 2;       // 阶数
type = 'low';  // 低通
```

#### Step 2 — 归一化频率

数字滤波器设计的核心是**频率归一化**。MATLAB 的 `butter()` 要求
截止频率以 **Nyquist 频率为单位**：

```
Nyquist = fs / 2 = 25000 Hz
归一化截止 = fc / Nyquist = 200 / 25000 = 0.008
```

这就是 `butter(2, 200/25000)` 中第二个参数的含义。

> 归一化是因为 z 平面的单位圆对应 0 到 Nyquist 频率。
> 设计 IIR 的本质是在 z 平面上放置极点和零点以达到期望的幅频响应。

#### Step 3 — 运行 MATLAB

```matlab
fs = 50000;
fc = 200;
[b, a] = butter(2, fc / (fs / 2), 'low');
```

`butter()` 内部做了这些事情：
1. 设计模拟 Butterworth 原型（s 平面上的极点）
2. 用双线性变换（bilinear transform）映射到 z 平面
3. 返回数字滤波器的分子系数 `b` 和分母系数 `a`

其中**双线性变换**是关键步骤：

```
s = 2 x fs x (1 - z^(-1)) / (1 + z^(-1))
```

它将 s 平面的整个虚轴压缩映射到 z 平面的单位圆上，避免了
模拟到数字转换中的混叠问题。代价是频率轴的非线性扭曲
（频率弯折/warping），`butter()` 内部自动做了预矫正。

#### Step 4 — 系数的含义

MATLAB 返回：
```
b = [b0, b1, b2]    // 分子系数（前馈）
a = [ 1, a1, a2]    // 分母系数（反馈）, a0 恒为 1
```

对应的**传递函数**（z 域）：

```
             b0 + b1.z^(-1) + b2.z^(-2)
H(z) = -----------------------------------
            1  + a1.z^(-1) + a2.z^(-2)
```

展开为**差分方程**（时域，Direct Form I）：

```
y[n] = b0.x[n] + b1.x[n-1] + b2.x[n-2] - a1.y[n-1] - a2.y[n-2]
```

这 5 个系数（b0,b1,b2,a1,a2）完全定义了滤波器的所有行为。
每个采样周期只需要 **5 次乘法 + 4 次加法**。

> 注意差分方程中 a1, a2 前面是负号。这是因为传递函数分母是
> 1 + a1.z^(-1) + a2.z^(-2)，移项后 y[n] 项变号。
> MATLAB 的 a1=-1.96446 意味着差分方程里是 -(-1.96446).y[n-1] = +1.96446.y[n-1]。

#### Step 5 — 验证

`biquad_design.m` 使用 `freqz()` 计算频率响应，输出关键频点的衰减值。
验证要点：
- fc 处是否真的 -3dB ✓（当前 200Hz 处 -3.01dB）
- 通带内是否平坦 ✓（当前 10~100Hz 都在 -0.3dB 以内）
- PWM 频率处是否充分衰减 ✓（理论 >80dB）

#### Step 6 — 写入 C 代码

```c
// I.c: 系数宏
#define BIQUAD_B0  0.0001551484f
#define BIQUAD_B1  0.0003102968f
#define BIQUAD_B2  0.0001551484f
#define BIQUAD_A1  (-1.9644605802f)
#define BIQUAD_A2  0.9650811739f

// I.c: Direct Form I 实现（每个采样周期）
float fIU;
fIU = BIQUAD_B0 * (float)i16IU_mA      // b0.x[n]
    + BIQUAD_B1 * s_fX1U               // b1.x[n-1]
    + BIQUAD_B2 * s_fX2U               // b2.x[n-2]
    - BIQUAD_A1 * s_fY1U               // -a1.y[n-1]
    - BIQUAD_A2 * s_fY2U;              // -a2.y[n-2]
// 更新历史
s_fX2U = s_fX1U; s_fX1U = (float)i16IU_mA;
s_fY2U = s_fY1U; s_fY1U = fIU;
```

**状态变量**（每相 4 个 float）：
| 变量 | 含义 |
|------|------|
| `s_fX1` | x[n-1]：上一拍的输入 |
| `s_fX2` | x[n-2]：上两拍的输入 |
| `s_fY1` | y[n-1]：上一拍的输出 |
| `s_fY2` | y[n-2]：上两拍的输出 |

> 这些状态变量就是"滤波器的记忆"——它们让滤波器"知道"过去的信号是什么样的，
> 从而推算当前信号中哪些是真实变化、哪些是噪声。

### 5.6 为什么是 2 阶而不是 3 阶或更高

| 阶数 | 衰减 | 状态变量 | 建立时间 | 实际收益 |
|------|------|------|------|------|
| 1 (EMA) | -20 dB/dec | 1/相 | ~6ms | PWM 衰减 46dB，不够 |
| **2 (当前)** | **-40 dB/dec** | **4/相** | **~12ms** | **PWM 衰减 84dB，已经低于 ADC 1LSB** |
| 3 | -60 dB/dec | 6/相 | ~18ms | PWM 衰减 126dB，但 ADC 已经分辨不出 |
| 4 | -80 dB/dec | 8/相 | ~24ms | 过度设计，延迟翻倍 |

**2 阶是甜点**：PWM 纹波已经被压到 ADC 分辨率以下（<1 LSB），
继续加阶数只会增加延迟和内存，不会进一步改善可见的波形质量。

## 6. 设计原理总结

### 6.1 为什么不选其他方案

### 为什么不用 FIR

| | IIR (Biquad) | FIR |
|------|:---:|:---:|
| 等效衰减所需阶数 | 2 | ~200+ |
| 状态变量 | 4/相 | 200+/相 |
| 延迟 | ~3ms | ~20ms+ |
| ISR 计算量 | ~30 FPU 周期 | ~200+ 周期 |
| 线性相位 | 无 | 有 |
| 显示用途需要线性相位？ | **不需要** | — |

### 为什么不用 CMSIS-DSP

HC32F460 项目仅包含 CMSIS Core 头文件，未包含 CMSIS-DSP 库（`.c`/`.a`）。手写 28 行 Biquad Direct Form I 完全等价，且编译器能直接内联优化。

### 为什么不用整数 Q8 实现（像旧 EMA 那样）

旧 EMA 用整数 Q8 是因为 1 阶 IIR 系数简单（alpha=8/256 刚好是 2 的幂）。
Butterworth 系数是任意的浮点数，用整数 Q8 近似会引入量化误差，
可能导致极点偏移、滤波器不稳定。Cortex-M4F 有硬件 FPU，
float32 乘加和整数一样快（单周期），没有理由不用 float。

### 6.2 各滤波器的核心差异

```
               通带          过渡带        阻带(PWM)
EMA(1阶):     ~~~~~~~~\___________________  -20dB/dec, 46dB@25kHz
Biquad(2阶):  ~~~~~~~~~~~~~\______________  -40dB/dec, 84dB@25kHz
Chebyshev:    ~~~~∿∿∿∿~~~~\_____________  通带纹波会扭曲电流波形
FIR(200阶):   ~~~~~~~~~~~~~~~|~~~~~~~~~~~~  完美的线性相位，但 200 阶太贵
```

Butterworth Biquad 在这个场景是性价比最高的：足够平坦的通带 + 足够陡的衰减 + 极低的计算量。

## 8. 调试建议

### J-Scope

- `g_i_iu_disp`: 滤波后 x10 + 10000, 零电流 = 10000。表达式 `(g_i_iu_disp - 10000) / 10` 得到 mA
- `g_i_iu_filt`: Q8 格式, 表达式 `g_i_iu_filt / 256` 得到 mA
- `g_i_iu_ma`: 瞬时 mA, 可对比滤波前后的纹波差异

### VOFA+

- 发送率: **1 kHz** (1ms 间隔)
- 协议: JustFloat
- 数据: `(disp - 10000) / 10` mA -> `x0.001` -> A
- 波形干净度不如 J-Scope 的原因: 1kHz 下采样 vs J-Scope 的 HSS 直接内存读取
- 如需继续优化 VOFA+ 显示质量，可在 main.c 发送前加滑动平均

### 修改截止频率

1. 编辑 `biquad_design.m` 中的 `fc` 值
2. 在 MATLAB 中运行，获取新系数
3. 更新 `I.c` 中的 `BIQUAD_B0` ~ `BIQUAD_A2` 宏
4. 重新编译
