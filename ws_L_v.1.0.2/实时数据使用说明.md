# 实时数据使用说明

## 概述

实时数据（`AppRealTimeData_t`）用于存储设备运行时的动态数据，**不保存到 Flash**，仅运行时存在内存中。其他设备模块（如霍尔传感器、ADC、故障检测等）可以通过以下接口修改实时数据，Modbus 协议会自动读取这些数据供上位机查询。

## 实时数据寄存器映射

| 寄存器地址 | 参数名 | 类型 | 单位 | 说明 |
|-----------|--------|------|------|------|
| 0x2730 | real_speed | int16_t | r/min | 实时转速 |
| 0x2731 | real_angle | int16_t | 0.1° | 实时角度 |
| 0x2732 | real_voltage | uint16_t | 0.1V | 电压 |
| 0x2733 | real_current | uint16_t | 1mA | 电流 |
| 0x2737 | real_direction | int16_t | - | 实时转向 |
| 0x2740 | fault_status | uint16_t | - | 故障状态（位标志） |

## 故障状态位定义

| 位 | 宏定义 | 说明 |
|---|--------|------|
| bit0 | FAULT_BIT_OVERVOLTAGE | 过压 |
| bit1 | FAULT_BIT_OVERCURRENT | 过流 |
| bit2 | FAULT_BIT_OVERTEMP | 过温 |
| bit3 | FAULT_BIT_RESET | 复位 |
| bit4 | FAULT_BIT_OVERLOAD | 过载 |
| bit5 | FAULT_BIT_STALL | 堵转 |

## 修改实时数据的三种方式

### 方式一：直接操作全局变量（最简单）

```c
#include "App_Params.h"

// 在霍尔传感器更新时
g_RealTimeData.real_speed = hall_speed;      // 单位 r/min
g_RealTimeData.real_angle = hall_angle;      // 单位 0.1°
g_RealTimeData.real_direction = hall_dir;

// 在 ADC 采集完成后
g_RealTimeData.real_voltage = adc_voltage;   // 单位 0.1V
g_RealTimeData.real_current = adc_current;   // 单位 1mA
```

### 方式二：使用 RealTime_Update() 函数（推荐，可选择性更新）

```c
#include "App_Params.h"

// 只更新转速，其他参数保持不变
int16_t speed = 1500;
RealTime_Update(&speed, NULL, NULL, NULL, NULL);

// 只更新电压和电流
uint16_t volt = 320;  // 32.0V
uint16_t curr = 1500; // 1.5A
RealTime_Update(NULL, NULL, &volt, &curr, NULL);

// 一次性更新所有参数
int16_t spd = 1200;
int16_t ang = 450;
uint16_t vlt = 310;
uint16_t cur = 2000;
int16_t dir = 1;
RealTime_Update(&spd, &ang, &vlt, &cur, &dir);
```

### 方式三：故障位操作（内联函数，零开销）

```c
#include "App_Params.h"

// 设置故障位（按位或）
RealTime_SetFault(FAULT_BIT_OVERVOLTAGE);   // 设置过压故障
RealTime_SetFault(FAULT_BIT_STALL);          // 设置堵转故障
RealTime_SetFault(FAULT_BIT_OVERCURRENT | FAULT_BIT_OVERTEMP);  // 同时设置多个

// 清除故障位（按位与取反）
RealTime_ClearFault(FAULT_BIT_OVERVOLTAGE); // 清除过压故障

// 检查故障位
if (RealTime_CheckFault(FAULT_BIT_STALL))
{
    // 处理堵转故障
}

if (RealTime_CheckFault(FAULT_BIT_OVERCURRENT))
{
    // 处理过流故障
}
```

## 典型使用场景

### 场景1：霍尔传感器更新转速和角度

在 `dev_hall.c` 或 `dev_motor_hall.c` 中：

```c
#include "App_Params.h"

void Hall_UpdateCallback(int16_t speed, int16_t angle, int16_t direction)
{
    RealTime_Update(&speed, &angle, NULL, NULL, &direction);
}
```

### 场景2：ADC 更新电压和电流

在 `dev_adc.c` 或 `dev_voltage.c` 中：

```c
#include "App_Params.h"

void Adc_UpdateCallback(uint16_t voltage, uint16_t current)
{
    RealTime_Update(NULL, NULL, &voltage, &current, NULL);
}
```

### 场景3：故障检测

在故障检测模块中：

```c
#include "App_Params.h"

void Fault_Check(void)
{
    // 过压检测
    if (g_RealTimeData.real_voltage > g_AppParam.voltage_upper_limit)
    {
        RealTime_SetFault(FAULT_BIT_OVERVOLTAGE);
    }
    else
    {
        RealTime_ClearFault(FAULT_BIT_OVERVOLTAGE);
    }

    // 过流检测
    if (g_RealTimeData.real_current > g_AppParam.current_upper_limit)
    {
        RealTime_SetFault(FAULT_BIT_OVERCURRENT);
    }
    else
    {
        RealTime_ClearFault(FAULT_BIT_OVERCURRENT);
    }

    // 堵转检测（转速为0但有目标转速）
    if ((g_RealTimeData.real_speed == 0) && (g_AppParam.target_speed != 0))
    {
        RealTime_SetFault(FAULT_BIT_STALL);
    }
    else
    {
        RealTime_ClearFault(FAULT_BIT_STALL);
    }
}
```

## 调试

在 `App_Params.h` 中有两个调试宏：

```c
// 取消注释：每5秒通过 RTT 打印一次实时数据
#define APP_PARAMS_REALTIME_DBG

// 取消注释：每5秒各实时参数 +1，用于测试上位机读取
#define APP_PARAMS_REALTIME_SIMULATE
```

## 注意事项

1. 实时数据**不保存到 Flash**，掉电后丢失
2. 所有实时数据都是**只读**的（从 Modbus 协议角度），上位机只能读取不能写入
3. 故障位支持同时设置多个，使用 `|` 组合宏定义即可
4. `RealTime_SetFault` 和 `RealTime_ClearFault` 是内联函数，无函数调用开销
