# 六步方波换相 — Hall 传感器 + SDH21263 预驱

## 硬件连接

### Hall 传感器 (3 路, 120° 电角度布置)

| 通道 | GPIO | EXTINT | IRQ | bit 权重 |
|------|------|--------|-----|---------|
| Hall U (HA) | PA10 | EXTINT_CH10 | INT010_IRQn | 0x04 |
| Hall V (HB) | PA9  | EXTINT_CH09 | INT009_IRQn | 0x02 |
| Hall W (HC) | PA8  | EXTINT_CH08 | INT008_IRQn | 0x01 |

三路 GPIO 拼为 3-bit 状态码: `state = (HA<<2) | (HB<<1) | HC`, 范围 0~7, 其中 0 和 7 为非法状态。

### PWM 输出 (TMR4 Unit 3, 50kHz 中心对齐)

| 通道 | 高侧 | 低侧 | GPIO Func2 |
|------|------|------|------------|
| U | PB9 (OUH) | PB8 (OUL) | TIM4_3_OUH / OUL |
| V | PB7 (OVH) | PB6 (OVL) | TIM4_3_OVH / OVL |
| W | PB5 (OWH) | PB4 (OWL) | TIM4_3_OWH / OWL |

### SDH21263 预驱真值表 (每相)

| H 输入 | L 输入 | 输出 |
|--------|--------|------|
| HIGH | HIGH | 上管 ON |
| LOW  | LOW  | 下管 ON |
| HIGH | LOW  | 互锁 -> 两管均 OFF |
| LOW  | HIGH | 互锁 -> 两管均 OFF |

## 霍尔状态与扇区

### 实测霍尔序列

顺时针 (CW) 拧电机轴:
```
0x02 -> 0x06 -> 0x04 -> 0x05 -> 0x01 -> 0x03 -> 0x02
```

逆时针 (CCW) 拧电机轴:
```
0x01 -> 0x05 -> 0x04 -> 0x06 -> 0x02 -> 0x03 -> 0x01
```

### 扇区定义

| Hall (HA HB HC) | 二进制 | 扇区 | 中心电角度 |
|-----------------|--------|------|-----------|
| 1 0 1 | 5 (0x05) | 扇区 1 | 60° |
| 1 0 0 | 4 (0x04) | 扇区 2 | 120° |
| 1 1 0 | 6 (0x06) | 扇区 3 | 180° |
| 0 1 0 | 2 (0x02) | 扇区 4 | 240° |
| 0 1 1 | 3 (0x03) | 扇区 5 | 300° |
| 0 0 1 | 1 (0x01) | 扇区 6 | 0°/360° |
| 0 0 0 | 0 | 非法 | — |
| 1 1 1 | 7 | 非法 | — |

### 转向与角度关系

- **CW 旋转**：扇区号递减 (4->3->2->1->6->5), 电角度递减 (240°->180°->120°->...)
- **CCW 旋转**：扇区号递增 (6->1->2->3->4->5), 电角度递增 (0°->60°->120°->...)

这是本电机的特殊性：CW 物理旋转对应递减电角度。标准电机通常是 CW=递增。此差异影响换相表的正反转分配。

## 电压矢量与换相步

### 六步换相步定义 (dev_commutation.c)

每步导通两个 MOSFET, 第三相浮空:

| 换相步 | 导通 | 开关状态 (UH UL VH VL WH WL) | 电压矢量 | 定子磁场角度 |
|--------|------|------------------------------|---------|-------------|
| 0 | UH + VL | 1 0 0 0 0 1 | V6 | 330° |
| 1 | UH + WL | 1 0 0 0 1 0 | V1 | 30° |
| 2 | VH + WL | 0 0 1 0 1 0 | V2 | 90° |
| 3 | VH + UL | 0 1 1 0 0 0 | V3 | 150° |
| 4 | WH + UL | 0 1 0 0 1 0 | V4 | 210° |
| 5 | WH + VL | 0 0 0 1 1 0 | V5 | 270° |

TMR4 PWM 模式:
- **导通相 (高侧)**: SYNC 模式, 占空比 = 目标值. H=L=PWM, 上管斩波.
- **导通相 (低侧)**: SYNC 模式, 占空比 = 2%. H=L~LOW, 下管常通.
- **浮空相**: COMPLEMENTARY 模式, 占空比 = 50%. H!=L -> SDH21263 互锁 -> 两管均 OFF.

### 换相规则

forward_map: 扇区中心角度 **+90°**, 选最接近的电压矢量.
reverse_map: 扇区中心角度 **-90°**, 选最接近的电压矢量.

### 本电机的特殊性：表名与实际方向相反

由于 CW 旋转 = 递减电角度, "+90°"在递减方向上实际上是滞后而非超前. 因此:

- **CW 模式 (CLOSED_FW)** 需使用 **reverse_map** (sector-90°) — 在递减方向上 -90° 才是超前
- **CCW 模式 (CLOSED_RV)** 需使用 **forward_map** (sector+90°) — 在递增方向上 +90° 才是超前

### Hall -> 换相步 映射表 (dev_comm_runner.c)

```c
// CW 模式用 reverse_map (sector -90°)
s_hall2step_cw[8]  = {0xFF, 5, 3, 4, 1, 0, 2, 0xFF};

// CCW 模式用 forward_map (sector +90°)
s_hall2step_ccw[8] = {0xFF, 2, 0, 1, 4, 3, 5, 0xFF};
```

#### CW 表验证 (Hall 0x02 -> 0x06 -> 0x04 -> 0x05 -> 0x01 -> 0x03)

使用 reverse_map (sector -90°):

| Hall | 扇区 | -90° | 电压矢量 | 换相步 | 定子磁场 |
|------|------|------|---------|--------|---------|
| 0x02 | 4 (240°) | 150° | V3 | 3 | 150° |
| 0x06 | 3 (180°) | 90°  | V2 | 2 | 90° |
| 0x04 | 2 (120°) | 30°  | V1 | 1 | 30° |
| 0x05 | 1 (60°)  | 330° | V6 | 0 | 330° |
| 0x01 | 6 (0°)   | 270° | V5 | 5 | 270° |
| 0x03 | 5 (300°) | 210° | V4 | 4 | 210° |

步序列: **3 -> 2 -> 1 -> 0 -> 5 -> 4** (REVERSE). 定子磁场递减 (150°->90°->30°->330°->270°->210°), 与 CW 转子方向一致, 始终比转子超前 90°.

#### CCW 表验证 (Hall 0x01 -> 0x05 -> 0x04 -> 0x06 -> 0x02 -> 0x03)

使用 forward_map (sector +90°):

| Hall | 扇区 | +90° | 电压矢量 | 换相步 | 定子磁场 |
|------|------|------|---------|--------|---------|
| 0x01 | 6 (0°)   | 90°  | V2 | 2 | 90° |
| 0x05 | 1 (60°)  | 150° | V3 | 3 | 150° |
| 0x04 | 2 (120°) | 210° | V4 | 4 | 210° |
| 0x06 | 3 (180°) | 270° | V5 | 5 | 270° |
| 0x02 | 4 (240°) | 330° | V6 | 0 | 330° |
| 0x03 | 5 (300°) | 30°  | V1 | 1 | 30° |

步序列: **2 -> 3 -> 4 -> 5 -> 0 -> 1** (FORWARD). 定子磁场递增 (90°->150°->210°->270°->330°->30°), 与 CCW 转子方向一致, 始终比转子超前 90°.

## 中断触发后函数调用链

```
硬件: PA10/PA9/PA8 任一 GPIO 跳变 (双边沿触发)
  -> EXTINT 硬件滤波 (DIV64, ~1.56MHz, 滤除 <~10us 毛刺)
  -> NVIC 触发 ISR

ISR: hall_u_isr() / hall_v_isr() / hall_w_isr()
  -> hall_common_handler(ch)                       [hall_sensor_3ch.c]
      1. EXTINT_ClearExtIntStatus()               — 清除中断标志
      2. read_hall_state_raw()                    — 读三路 GPIO, 拼 3-bit 状态码
      3. state == last_hall_state ? -> return      — 状态未变 = 振铃, 丢弃 (不调 GetDelta)
      4. Timer6_Timebase_GetDelta()               — 获取距上次 GetDelta 的时间间隔, 重置计数器
      5. Timer6_Timebase_DeltaToUs()              — 转换为微秒
      6. interval < 50us ? -> return (noise)       — 间隔检查, 拦截噪声
      7. hall_to_step[state]                      — 查换相表, 得换相步 (0~5)
      8. step == 0xFF ? -> on_fault -> return      — 非法状态 (000/111)
      9. state != RUNNING ? ->                    — 对齐/空闲阶段:
           记录 state, pulse_counter++, 打印 [HALL] IDLE/ALIGN, return
     10. diff = (step - last_step + 6) % 6        — 相邻性检查
         diff != 1 && diff != 5 ? ->               — 非法跳变 (diff=2/3/4):
           打印 [HALL] BAD, return                 — 丢弃且不更新 last_step
     11. 打印 [HALL] COMM (每 8 次合法换相)         — 显示 Hall/old_step->new_step/interval
     12. 更新 last_hall_state, last_step, pulse_counter++
     13. ISR 内调用换相回调 (带耗时测量):

runner_on_hall_step(step, dir)                    [dev_comm_runner.c]
  -> Commutation_Step(step, pwm_freq_hz, duty)   [dev_commutation.c]
      1. 钳位 duty 到 2%~98%
      2. 频率变化? -> TMR4_PWM_SetFrequency() + 使 mode 缓存失效
      3. 遍历 3 通道 (U/V/W):
         - 通道模式变化 (SYNC<->COMP) -> TMR4_PWM_SetChannelMode(ch, mode, duty)
         - 同模式仅占空比变化 -> TMR4_PWM_SetDutyFloat(ch, duty)
         - 无变化 -> 跳过 (懒更新优化)

TMR4_PWM_SetChannelMode(ch, mode, duty)          [tmr4_pwm.c]
  -> 重新初始化该通道的 OC + PWM 模块
  -> COMPLEMENTARY: 使能低侧 OC, 禁止高侧 OC, DEAD_TMR 模式
  -> SYNC: 使能高低侧 OC, THROUGH 模式
  -> 写入影子寄存器 (OCCR, OCMR) + 配置影子传输条件为 PEAK
  -> 计数器 PEAK 时 3 通道同步传输 -> 无毛刺换相
```

## 噪声分层防御与已知问题

### 5 层防御

| 层 | 位置 | 机制 | 拦截对象 |
|----|------|------|---------|
| 1 | 硬件 | EXTINT 滤波器 DIV64 (~1.56MHz) | <~10us 脉冲毛刺 |
| 2 | ISR step 3 | state == last_hall_state -> 返回, 不调 GetDelta | 同一跳变沿的振铃 |
| 3 | ISR step 6 | 脉冲间隔 < 50us -> 丢弃 | 穿透硬件滤波的短脉冲 |
| 4 | ISR step 8 | Hall = 000 或 111 -> on_fault | 三路同时被噪声拉高/拉低 |
| 5 | ISR step 10 | step diff 必须为 ±1 -> 否则 BAD+return | 噪声产生的随机合法 Hall 状态 |

### 已知问题：PWM 噪声同时触发三个 ISR

**现象**：闭环运行时，`baddiff` 持续增长. 每次 PWM 开关噪声爆发, 三个 Hall 的 EXTINT 几乎同时触发, 产生一组三条 BAD:

```
BAD ch=1 Hall=0x04 step=3->1 diff=4 int=~200   (HB 线被噪声触发)
BAD ch=2 Hall=0x05 step=3->0 diff=3 int=~250   (HC 线被噪声触发)
BAD ch=0 Hall=0x01 step=3->5 diff=2 int=~600   (HA 线被噪声触发)
```

**分析**：三个 ISR 读到的 GPIO 是噪声产生的随机中间态. 当前 CW 表 (sector-90°) 对这些状态的映射都是正确的——它们的 step 与 last_step 都不邻接 (diff=2/3/4), 被第 5 层全部拦截. **BAD 证明换相表在正确工作, 不是表错了.**

**影响**：每次噪声爆发产生 3 个 BAD, 积累为 `baddiff/valid ≈ 13%`. 噪声不直接影响换相 (BAD 被拦截, last_step 不更新), 但三个 ISR 接连执行消耗 CPU、污染 GetDelta 计数器.

**待尝试方案**：ISR 重入锁 — 第一个 ISR 设忙标志, 100us 内后续 ISR 直接返回. 可将每次噪声爆发的 3 个 BAD 减少为 1 个.

## 运行模式 (CommRunner)

| 模式 | 行为 |
|------|------|
| STOP (0) | 3 相全 98% SYNC — 上管全通 = 滑行/刹车 |
| OPEN_FW (1) | 纯开环正转, 定时器按设定间隔触发 Commutation_Step |
| OPEN_RV (2) | 纯开环反转 |
| CLOSED_FW (3) | 开环斜坡加速 -> 飞行启动 -> Hall ISR 接管, 闭环正转 |
| CLOSED_RV (4) | 开环斜坡加速 -> 飞行启动 -> Hall ISR 接管, 闭环反转 |

闭环启动流程:
1. `CommRunner_SetMode(CLOSED_FW/RV)` -> 停 Hall, 启动开环 ramp
2. 开环: `open_loop_tick()` 每主循环调用一次, 按线性斜坡计算间隔, 定时触发 Commutation_Step
3. 斜坡结束 -> `hall_3ch_set_table()` 加载方向对应的换相表
4. `hall_3ch_start_flying()`: 读 Hall GPIO 双采样 -> 查表得当前步 -> 立即换相 -> 切 RUNNING
5. 此后 Hall ISR 自动接管换相, 主循环仅做 RPM 计算和堵转检测

## M 法转速计算

- ISR 中: 每次合法脉冲 `pulse_counter++`
- 主循环 `hall_3ch_update()` 中:
  1. `pulse_delta = pulse_counter - last_pulse_count`
  2. `elapsed = now - last_rpm_update_us`
  3. 条件: `elapsed >= 20ms && pulse_delta >= 6` 时:
     `RPM = pulse_delta * 60000000 / (elapsed * pole_pairs * 6)`
  4. 6 样本滑动窗口滤波 -> `RPM_filtered`
  5. 超时: `elapsed > 500ms && pulse_delta == 0` -> RPM=0 (堵转)

## 调试经验总结

1. **单表不能用于双方向**：最初尝试用一张标准表同时驱动 CW 和 CCW, 但 CW/CCW 的定子磁场超前方向相反, 单表无法同时满足. 必须正反转各一张表.
2. **forward_map/reverse_map 可能与实际模式名相反**：取决于电机的霍尔安装方向, "+90°"在递减角度方向上实际是滞后. 本电机 CW=递减角度, 所以 CW 模式用 reverse_map (sector-90°).
3. **三个独立 EXTINT 存在噪声同时触发问题**：PWM 开关噪声同时耦合到三路 Hall, 导致连续 BAD. 相邻性检查能拦截, 但无法消除根因.
4. **开环阶段电机可能反向转**：开环 FW 斜坡在本电机上实际驱动 CCW 方向, 导致 flying start 时转子方向与预期不符.
