# BLDC Six-Step Commutation: 30° vs 90° — Two Sides of the Same Coin

## The Question

When reading about BLDC Hall-sensor-based commutation, two numbers keep appearing:

- **90°** — the code sets the stator field +90° ahead of the rotor (for CW rotation)
- **30°** — many references (Microchip AN857, ST AN1946, etc.) talk about a "30° delay" after a Hall edge

Are these contradictory? Which one is correct? And can angles be arbitrarily defined anyway?

## Short Answer

**Both numbers are correct. They describe the same geometric relationship from two different implementation perspectives.**

The 90° is the **torque angle** (stator vs rotor, the fundamental physics). The 30° is the **time delay** from Hall edge to optimal commutation point, assuming Hall sensors are mounted at the "neutral" (winding-aligned) position. These two descriptions are mathematically equivalent.

---

## What the Code Actually Does

### Hall State → Rotor Angle Mapping (`hall_sensor_3ch.c`)

```
Hall 0x01 → 0°     Hall 0x05 → 60°
Hall 0x02 → 240°   Hall 0x06 → 180°
Hall 0x03 → 300°   Hall 0x00/0x07 → invalid (000 / 111)
Hall 0x04 → 120°
```

### Commutation Step → Stator Field Angle (`dev_commutation.c`)

| Step | High-Side | Low-Side | Field Angle |
|------|-----------|----------|-------------|
| 0 | UH | VL | 330° |
| 1 | UH | WL | 30° |
| 2 | VH | WL | 90° |
| 3 | VH | UL | 150° |
| 4 | WH | UL | 210° |
| 5 | WH | VL | 270° |

### CW Hall→Step Mapping (`dev_comm_runner.c`, `s_hall2step_cw`, labeled "sector -90deg"):

```
Hall 0x02 (rotor at 240°) → Step 3 → VH_UL → stator field at 150°
```

For CW rotation (decreasing angle): rotor at 240°, stator at 150°, the stator **leads** the rotor by **90°** in the direction of rotation.

---

## The Two Interpretations

### Interpretation A: 90° Torque Angle (what the code does)

```
          Stator field (150°)
              │
              │  90° torque angle
              ▼
     ←←←← Rotor (240°) ←←←←
           CW rotation
```

- The stator field vector is positioned **90 electrical degrees ahead** of the rotor
- Torque ∝ sin(angle between stator and rotor flux)
- sin(90°) = 1.0 → **maximum torque**
- As the rotor rotates through the 60° sector, the torque angle decreases from 90° to 30°, averaging ~0.955 of peak — this is standard for six-step (trapezoidal) commutation

**Implementation**: A lookup table that maps each Hall state directly to the stator step that produces the +90° offset. At every Hall edge, the commutation step changes immediately — no timer delay needed.

### Interpretation B: 30° Delay (what many app notes describe)

When Hall sensors are installed at the **neutral position** (aligned with phase windings), each Hall edge occurs at a **sector boundary** (0°, 60°, 120°, …). But optimal commutation happens at the **sector midpoint** (30°, 90°, 150°, …), which is 30° after the Hall edge.

```
  Hall edge (0°)      Optimal commutation (30°)
       │                      │
       ▼                      ▼
   ────●──────────────────●──────
       ←── 30° delay ──→
       (timer-based)
```

**Implementation**: A timer-based delay. At every Hall edge, start a timer for 30 electrical degrees, then commutate. This requires dynamic delay calculation (which varies with speed) and is more complex.

### Equivalence

| | "30° delay" method | Code's "90° offset" method |
|---|---|---|
| Trigger | Hall edge | Hall edge |
| Action | Wait 30°, then step to the "aligned" step | Immediately step to a step offset by +90° |
| Stator field angle at commutation | Rotor + 90° | Rotor + 90° |
| Net torque angle | 90° | 90° |
| Timer per step | Needed | Not needed |

**Same end result.** The code uses the lookup-table offset approach because it is simpler — no speed-dependent delay calculation required.

## Can Angles Be Arbitrarily Defined?

**Yes — the absolute angle coordinate system can be rotated arbitrarily.**

The absolute angle values (0°, 60°, 120°, …) assigned to Hall states and commutation steps have no physical significance on their own. They form a relative coordinate frame. What matters are two constraints:

### Constraint 1: Six equally-spaced states (60° apart)

Both the Hall states and the stator field steps must divide the 360° electrical cycle into six equal 60° sectors. This is a hard constraint imposed by the motor's pole-pair structure and three-phase geometry.

### Constraint 2: The Hall→Stator offset determines torque

For maximum torque, the stator field must lead the rotor (for CW) or lag the rotor (for CCW) by **90°** in the direction of rotation.

You could rotate the entire coordinate system by any arbitrary angle — say +45° to everything:

```
Hall 0x01: 0° → 45°
Step 0:   330° → 15°
```

As long as the **relative offset** between Hall and the corresponding stator step remains 90°, the motor behavior is identical. The absolute coordinate frame is a convention you set when labeling your states.

---

## What Happens With a 30° Offset?

If the Hall→Step mapping used sector-30° instead of sector-90°:

```
Hall 0x02 (rotor 240°) → Step at 210° (WH_UL) → torque angle = 240° - 210° = 30°
```

- sin(30°) = **0.5** → only 50% of maximum torque
- The motor would run weakly, with high current draw and poor efficiency

±90° is correct for maximum torque. ±30° would produce noticeably weak operation.

## 标定偏移量：为什么 CW 用 +5、CCW 用 +2？

### 标定公式

```c
// dev_comm_runner.c, calib_build_derived_tables()
g_calib_cw_table[h]  = (g_calib_table[h] + 5u) % 6u;   // CW 正转
g_calib_ccw_table[h] = (g_calib_table[h] + 2u) % 6u;   // CCW 反转
```

g_calib_table 是 mode 5 标定出的 0° 映射（磁场对准转子，零扭矩）。+5 和 +2 把零扭矩映射旋转为 ±90° 超前，产生最大扭矩。

### 偏移值与超前角的关系

```
偏移 1 步 = 超前  60° 电角度
偏移 2 步 = 超前 120° 电角度
偏移 3 步 = 超前 180° 电角度
偏移 4 步 = 超前 240° 电角度
偏移 5 步 = 超前 300° 电角度（等价于落后 60°）
```

```
Step 编号绕圆周:
  CCW(递增): 0 → 1 → 2 → 3 → 4 → 5 → 0
  CW(递减):  0 → 5 → 4 → 3 → 2 → 1 → 0

偏移含义:
  +2 = CCW 方向前进 2 步 = 超前 120° → CCW 旋转用
  +5 = CW 方向前进 1 步(-1 mod 6) = 超前 60° → CW 旋转用
```

### 为什么 CW 偏 60° 而 CCW 偏 120°？扭矩不一样吗？

理想正弦马达扭矩 = sin(超前角)：

```
sin(60°)  = 0.866
sin(120°) = 0.866
```

两个 sin 值相等，但实际 BLDC 的扭矩-角度曲线不是正弦波，是**梯形波**：

```
扭矩
 ↑    ┌──────────────────┐
 │    │   平坦区           │
 │    │  ~80° ── ~130°    │  ← 扭矩最大且稳定
 │   ╱                    ╲
 │  ╱                      ╲
 │ ╱                        ╲
─┴──────────────────────────────→ 超前角
 0°    60°     90°     120°   180°
       ↑       ↑        ↑
    CW(+5)   理想值   CCW(+2)
    (60°)           (120°)
```

+2（120°超前）落在梯形波平坦区，扭矩最大。+5（60°超前）落在斜坡上。所以 CW 方向用 +5 转速较低，CCW 方向用 +2 转速较高。

### 为什么 CW 不也用 +2（120°超前）？

CW 旋转方向是 step 编号递减的（0→5→4→3→2→1）。"提前 2 步"在递减序列里是 `(step - 2) mod 6` = `(step + 4) mod 6`。如果 CW 也用 120° 超前，偏移应该是 +4 而不是 +5。

这就是为什么试验中：
- `x=5, y=1`（CW超前60°, CCW超前60°）→ 转速低，两个方向都没进平坦区
- `x=4, y=2`（CW超前120°, CCW超前120°）→ 转速高，两个方向都在平坦区
- `x=3, y=3`（都超前180°）→ 转速也高，因为梯形波在 180° 附近还有一个上升沿，实际有效超前角因电流滞后而落在平坦区内

### 换控制板/换电机后如何确定偏移值？

`+5` 和 `+2` 由**控制板硬件**决定（6288T-MNS 真值表 + TMR4 接线 + 六步通电表），换电机不需要改。

如果换了控制板（不同驱动芯片、不同接线），验证方法：

1. 跑 mode 5 标定得到 g_calib_table
2. 在代码里改 `calib_build_derived_tables()` 中的偏移值
3. 分别跑 mode 6（CW）和 mode 7（CCW）
4. 看哪个偏移值下转速最高、运行最平稳
5. 那个就是正确的偏移

规则：**偏移值应使磁场超前转子落在 80°~130° 的梯形波平坦区内。**

## The Motor-Specific Calibration

The comment block in `dev_comm_runner.c` (lines 14–21) documents that the Hall→Step mapping was **measured and calibrated for this specific motor**:

```
Physical Hall sequence (measured on this motor):
  CW:  0x02 -> 0x06 -> 0x04 -> 0x05 -> 0x01 -> 0x03  (decreasing sector#)
  CCW: 0x01 -> 0x05 -> 0x04 -> 0x06 -> 0x02 -> 0x03  (increasing sector#)

For this motor, CW = decreasing angle = needs sector-90deg (reverse_map).
CCW = increasing angle = needs sector+90deg (forward_map).
So CW mode uses reverse_map, CCW mode uses forward_map — they are SWAPPED
relative to their names because of the motor's Hall mounting orientation.
```

Key points:

1. **The mappings are not generic formulas.** They reflect this motor's actual Hall sensor mounting orientation relative to the rotor magnets.
2. **`forward_map` and `reverse_map` are swapped** relative to their names — a consequence of the physical Hall mounting.
3. **Different motors may need different tables** even though the 90° torque-angle principle is universal.

## Summary

| Aspect | "30°" (app note style) | "90°" (the code's style) |
|---|---|---|
| What it describes | Time delay from Hall edge to commutation point | Stator-rotor magnetic field angle |
| Physical meaning | Timing / phase advance | Torque-producing angle |
| Impact on torque | Indirect (via torque angle) | Direct (sin θ) |
| Implementation | Timer-based delay | Lookup-table offset |
| Complexity | Higher (speed-dependent) | Lower (static table) |

**Both numbers are correct. They are two implementation perspectives on the same geometric requirement: the stator magnetic field must lead (or lag, for reverse) the rotor by 90 electrical degrees in the direction of rotation to produce maximum torque.**

If the motor runs with good torque and efficiency, the Hall calibration table is correct.
