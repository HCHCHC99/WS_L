# 预驱芯片对比：SDH21263 vs 6288T-MNS

## 真值表

### SDH21263

| H | L | HO | LO | 状态 |
|---|---|----|----|------|
| H | H | H | L | **上管导通** |
| L | L | L | H | **下管导通** |
| H | L | L | L | 互锁关断 |
| L | H | L | L | 互锁关断 |

- **导通条件**：H = L（同相）
- **关断条件**：H ≠ L（差分）
- **上/下管区分**：同相时占空比高低决定（高占空比→上管，低占空比→下管）

### 6288T-MNS

| HIN | LIN | HO | LO | 状态 |
|-----|-----|----|----|------|
| L | L | L | L | 两管关断 |
| L | H | L | H | **下管导通** |
| H | L | H | L | **上管导通** |
| H | H | L | L | 两管关断 |

- **导通条件**：H ≠ L（差分）
- **关断条件**：H = L（同相）
- **上/下管区分**：差分时谁高谁低决定（H=H/L=L→上管，H=L/L=H→下管）

---

## 核心差异

| 维度 | SDH21263 | 6288T-MNS |
|------|----------|-----------|
| 导通逻辑 | **同相导通**（H=L 则 ON） | **差分导通**（H≠L 则 ON） |
| 关断逻辑 | **差分关断**（H≠L 则 OFF） | **同相关断**（H=L 则 OFF） |
| 直通保护 | H≠L 自动互锁 | H=H 或 H=L 均关断（同相即关） |
| 输入语义 | L=ON, H=ON 由占空比区分 | L, H 的电平组合直接决定 |

**逻辑完全相反。** 同一个 MCU 输出到两个芯片，导通/关断行为对调。

---

## MCU 驱动策略对比

### SDH21263 — 利用互补模式实现关断

```
上管导通：SYNC 模式, H=L=PWM(高占空比)
          → H=H 大部分时间 → 上管 ON

下管导通：SYNC 模式, H=L=PWM(低占空比)
          → L=L 大部分时间 → 下管 ON

浮空关断：COMPLEMENTARY 模式, H≠L
          → 硬件死区定时器自动产生互补 → H≠L 恒成立 → 互锁 OFF
```

- 使用 TMR4 的 **COMPLEMENTARY + DEAD_TMR** 模式产生互补信号
- 死区定时器自动保证 H 和 L 不重叠
- 导通时 H 和 L 输出相同波形（SYNC 模式）

### 6288T-MNS — 独立控制 H/L 实现差分

```
上管导通：HIGH_SIDE 模式
          → H OC 使能 = PWM, L OC 禁用 = LOW
          → PWM ON: H=H, L=L → 上管 ON
          → PWM OFF: H=L, L=L → 关断

下管导通：LOW_SIDE 模式
          → H OC 禁用 = LOW, L OC 使能 = HIGH
          → H=L, L=H → 下管 ON

浮空关断：OFF 模式
          → H OC 禁用 = LOW, L OC 禁用 = LOW
          → H=L, L=L → 关断
```

- 全部使用 TMR4 **THROUGH** 模式（不需要死区）
- 通过 OC 通道的**使能/禁用 + 独立占空比**控制 H 和 L
- 禁用时通过 `INVD_LOW` 把引脚强制拉到 LOW
- 上管斩波时 PWM OFF 期间两管均关断（体二极管续流）

## 代码层面改动对照

| 位置 | SDH21263 | 6288T-MNS |
|------|----------|-----------|
| `tmr4_pwm.h` 模式枚举 | `TMR4_MODE_SYNC` / `TMR4_MODE_COMPLEMENTARY` | `TMR4_MODE_OFF` / `TMR4_MODE_HIGH_SIDE` / `TMR4_MODE_LOW_SIDE` |
| `tmr4_pwm.c` SetChannelMode | 分支 SYNC(双路同占空比) / COMPLEMENTARY(低侧+死区) | 分支 OFF(双禁用) / HIGH_SIDE(H使能+L禁用) / LOW_SIDE(H禁用+L100%) |
| `tmr4_pwm.c` PWM 模式 | SYNC→THROUGH, COMP→DEAD_TMR | 全部 THROUGH |
| `dev_commutation.c` 状态表 | `M_SYNC` / `M_COMP` | `M_OFF` / `M_HIGH` / `M_LOW` |
| `dev_commutation.c` 关断 | COMPLEMENTARY + 50% 占空比 | OFF (双 LOW) |
| `dev_commutation.c` 导通 | SYNC + duty% / SYNC + 2% | HIGH_SIDE + duty% / LOW_SIDE(固定) |
| `dev_comm_runner.c` STOP | SYNC + 98% (上管全通刹车) | HIGH_SIDE + 98% (上管全通刹车) |
| 硬件死区 | 使用 | **不需要** |
| 引脚 INVD 电平 | 默认 LOW | LOW（禁用时强制拉低） |

## 刹车行为对比

| | SDH21263 | 6288T-MNS |
|------|----------|-----------|
| 刹车方式 | 3 相全 SYNC+98% → H=H=L=H → 三路上管全通 | 3 相全 HIGH_SIDE+98% → H=H/L=L → 三路上管全通 |
| 效果 | 三相短接至 DC+，动能消耗在绕组上 | 同左 |
| 滑行 | 3 相全 COMP+50% → H≠L → 三相关断 | 3 相全 OFF → H=L=LOW → 三相关断 |

两种芯片的 STOP 行为（刹车）**效果一致**，只是 MCU 输出的实现方式不同。

## 安全分析

### SDH21263 的安全特性
- H≠L（差分）自动互锁 → 上下管不会同时导通
- 即使 MCU 输出异常，差分状态下芯片硬保护

### 6288T-MNS 的安全特性
- H=L（同相）自动关断 → 上下管同时关
- MCU 只要保证 H 和 L 不同时为高/低（取决于导通意图），就不会意外导通
- 初始化时所有通道 OFF（H=L=LOW）→ 安全

### MCU 失控场景
| 场景 | SDH21263 | 6288T-MNS |
|------|----------|-----------|
| MCU 所有引脚 Hi-Z (复位) | 取决于上下拉电阻 | 取决于上下拉电阻 |
| MCU 所有引脚 HIGH (故障) | H=H → 三路上管全通（刹车） | H=H=L=H → 三路全关（滑行） |
| MCU 所有引脚 LOW (故障) | H=L → 三路下管全通（刹车）| H=L=L=L → 三路全关（滑行） |

**SDH21263 在 MCU 输出异常时可能意外导通（刹车），6288T-MNS 在 MCU 输出异常时默认关断（滑行）。** 从安全角度看，6288T-MNS 更倾向于关断（fail-safe），SDH21263 更倾向于刹车。

## 换芯片注意事项

1. **真值表完全相反**：不可直接替换，必须修改 MCU 驱动层
2. **不需要死区**：6288T-MNS 自带同相关断保护，MCU 侧不需要硬件死区
3. **制动策略相同**：都是用 HIGH_SIDE + 高占空比实现上管全通刹车
4. **校准参数不变**：`g_calib_table` 和偏移量（+5/+2）由 Hall 安装位置和 TMR4 接线决定，芯片更换不影响
5. **如果芯片再次更换**：只需修改 `tmr4_pwm.c` 的 `SetChannelMode` 和模式枚举，上层 `dev_commutation.c` / `dev_comm_runner.c` 通过模式抽象隔离了芯片差异
